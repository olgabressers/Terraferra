Alternative Business Model
Reverse-marketplace direction · visual style profiling · social discovery layered with AI. Three concept blocks plus a structural read of the reverse-marketplace competitor landscape.
Conclusion

The original 2022 model — a designer toolbox for pre-sales, crowdfunding, co-creation and ambassador commerce — stays as the foundation. Added in 2026: an ongoing marketplace layer (designer storefronts). On top of this, the alt-model proposes three additional discovery directions for the founder and CTO to evaluate: a style profile per user (A), a reverse marketplace where designers post briefs (B), and a personalised drop feed (C). The combination is not currently offered by any of the platforms surveyed (Farfetch, LTK, ShopMy, Rednote, TikTok, Pearpop, TRIBE, Cohley, JoinBrands, Brandbassador, Insense, Aspire, #paid, Trend).

Additional functionality · alt-model refocus
Additional operating model layers
A · Visual Style Profile
style profile per consumer / super fan / influencer — powers feed, drop alerts, brief matching
A style vector for each consumer, super fan and influencer (not designer). Must be defined for the profile to work — either automatically from on-platform activity (saves, pre-orders, votes) or from Instagram via a Section 03 compliant path, or manually from a swipe-quiz or tag selection at signup. Powers home-feed ranking, drop alerts, and fit-scoring for designer briefs.
B · Reverse Marketplace
designer posts a brief, influencers and super fans apply, AI ranks fit
Designer posts a brief (e.g. "knitwear drop next month, looking for 5 cottagecore influencers"). Influencers and super fans apply. AI ranks applications on audience fit, past sell-through and style overlap. Inverts today's model where designers hunt influencers manually.
C · Social Discovery × AI
personalised drop feed — replaces catalog browse; ranks drops on style × follow × drop momentum
This is how users find new drops on TF. Instead of browsing a catalog (Farfetch, Mytheresa) or searching, the viewer sees a personalised feed where each drop is ranked on three signals: Style (matches their taste profile from Block A), Follow (a designer or influencer they follow is involved), and Drop momentum (other users are pre-ordering, voting on, or sharing it). The drops come to the viewer, not the other way around — the algorithm pushes them, the viewer does not need to search. Search still exists as a secondary path; the main discovery mode is the feed. That is what makes TF social-commerce instead of a catalog marketplace.

The behavioural difference Farfetch shopper finds product through search and editorial. Rednote user finds product through algorithmic feed but with no buying loop. Positioning vs Existing Categories
Catalog marketplaces (Farfetch, Wolf & Badger, Not On The High Street) sell inventory through search and browse. Affiliate networks (LTK, ShopMy) attach creator links to existing products. Creator-marketing platforms (Pearpop, TRIBE, Cohley) run campaigns for Fortune-500 brands (named clients: Microsoft, Netflix, Mars, Unilever). None combine social discovery, visual AI and a reverse-marketplace structure inside one product, positioned for emerging affordable-luxury designers.

Recommendation · which to use for the MVP to find creators

Start on JoinBrands (free plan). It is the only option with zero fixed monthly cost — you pay just the 15% platform fee when you actually pay a creator, so an MVP can source and test creators with no subscription risk. Self-apply + brand-invite flow means creators come to the brief, and escrow + agreements are built in.

  • —Fastest fashion UGC: add Collabstr ($249/mo Pro) once you want instant, fixed-price fashion creator packages — free to browse the directory first before committing.
  • —Free but slower: Afluencer — post unlimited collabs free, but directory-based matching is slower; useful as a no-cost second funnel.
  • —Best fashion fit when budget allows: Insense — explicitly fashion-category, 80K+ vetted creators, UGC from $100/video; higher commitment than a free-tier MVP.
The enterprise platforms (Pearpop, TRIBE, Cohley, #paid, Aspire, Lefty) are demo-only / undisclosed pricing and built for Fortune-500 budgets — not MVP-appropriate.
04Instagram Style Scan — Options & Constraints
No consumer-facing fashion app currently runs visual AI on personal Instagram content for style profiling. The platforms in Sections 02 and 03 all use other signals. The blocker is technical, not competitive — explained below.
Key constraint No fully invisible legal path exists for personal Instagram accounts. Meta's APIs do not let third-party apps read a user's likes, saves or follows. Personal accounts — the majority of users — cannot expose any post content via API. The closest compliant path is OAuth + a manual photo upload at signup. Five compliant options below; A + B combined is the recommended pattern.
Option How it works Coverage UX time Risk / status
A · Instagram Login + manual upload Source: developers.facebook.com OAuth captures identity (username, profile photo). User then uploads 5 outfit photos. FashionCLIP builds the style vector from the uploads. All Instagram users — Low risk · Meta-sanctioned
B · Business / Creator API auto-pull Source: developers.facebook.com OAuth + Instagram Graph API reads the user's recent media (own posts only). Style vector auto-built. No likes / saves / follows accessible. Business + Creator accounts only — Low risk · Meta-sanctioned · app-review required
C · Public profile URL scrape User pastes their public Instagram profile URL. TF scrapes recent posts via headless browser or 3rd-party data provider. Public accounts only — High risk · Meta TOS violation · account & IP bans · prior litigation precedent
D · iOS / Android share extension User shares 5 Instagram posts to the TF mobile app via the OS share sheet. TF reads the shared image data directly. Mobile app users only — Low risk · user-initiated
E · Browser extension Chrome / Safari extension captures images as the user browses Instagram. Style vector built passively. Desktop users with extension installed Passive (over days) Medium risk · extension-store review · privacy positioning
Recommended pattern Combine A + B in a single flow. OAuth login first. If the connected account is Business or Creator, pull recent media via API and build the style vector in 5 seconds. If the account is Personal (the majority), prompt the user to upload 5 photos — same 30-second onboarding. D is an add-on for the iOS / Android apps. C is excluded on TOS grounds.
