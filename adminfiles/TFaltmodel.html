Alternative Business Model
Reverse-marketplace direction · visual style profiling · social discovery layered with AI. Three concept blocks plus a structural read of the reverse-marketplace competitor landscape.
01Three Concept Blocks
01 · Reverse Marketplace
Designer posts a brief. Ambassadors self-apply. AI ranks the fit.
Inverts the standard creator-discovery flow. Today, small fashion brands hire staff to find, negotiate with and brief influencers one-to-one. This bottleneck disappears when designers publish open briefs and ambassadors compete for selection.
Mechanism
  • Designer publishes drop brief: aesthetic, audience, deliverable, commission
  • Ambassadors browse open briefs, one-tap apply with their visual style profile attached
  • FashionCLIP scores style fit between ambassador's content and the brief lookbook
  • Designer reviews ranked applicants, accepts top matches
  • Removes negotiation overhead. Scales without adding headcount.
02 · Visual Style Profile
B2C visual AI. Built from 5 photos in 30 seconds.
Visual AI ported from B2B catalog enrichment into a B2C personalisation primitive. Every user — consumer, fan, ambassador — has a style vector. All downstream discovery, recommendation and brief-matching runs against it.
Mechanism
  • Onboarding: user uploads 5 outfit photos or pastes Instagram post URLs
  • FashionCLIP embeds each photo into a 512-dim style vector
  • Profile stored once. Used everywhere: feed ranking, drop notifications, brief matching
  • Instagram Graph API constrains "log-in and pull silently" — onboarding capture is the workable substitute
  • Re-trains opportunistically as the user saves / votes / pre-orders
03 · Social Discovery × AI
Three-axis personalisation in one product.
Combines visual style match (vector similarity), social proximity (follow graph) and drop velocity (community-voted pre-orders). Each axis exists separately on competing platforms; the combination was not observed in the platforms surveyed for the Comparison Matrix.
Axis split
  • Style match: FashionCLIP vector cosine similarity
  • Social proximity: who the user follows + who follows the user
  • Drop velocity: co-creation vote count + pre-order conversion rate
  • Each drop in the feed ranked by weighted sum across all three axes
02Instagram Style Scan — Options & Constraints
Meta's APIs do not expose a user's discovery signal (likes, saves, follows) to third parties. Personal accounts cannot share post content via API at all. The "log in, scan, done" experience exists only as a hybrid fallback. Five workable patterns:
Option How it works Coverage UX time Risk / status
A · Instagram Login + manual upload Source: developers.facebook.com OAuth captures identity (username, profile photo). User then uploads 5 outfit photos. FashionCLIP builds the style vector from the uploads. 100% of users ≈ 30 seconds Low risk · Meta-sanctioned
B · Business / Creator API auto-pull Source: developers.facebook.com OAuth + Instagram Graph API reads the user's recent media (own posts only). Style vector auto-built. No likes / saves / follows accessible. ≈ 5–10% of users (Business + Creator accounts only) ≈ 5 seconds Low risk · Meta-sanctioned · app-review required
C · Public profile URL scrape User pastes their public Instagram profile URL. TF scrapes recent posts via headless browser or 3rd-party data provider. ≈ 90% of users (public accounts) ≈ 10 seconds High risk · Meta TOS violation · account & IP bans · prior litigation precedent
D · iOS / Android share extension User shares 5 Instagram posts to the TF mobile app via the OS share sheet. TF reads the shared image data directly. Mobile app users only ≈ 30 seconds (multiple shares) Low risk · user-initiated
E · Browser extension Chrome / Safari extension captures images as the user browses Instagram. Style vector built passively. Desktop users with extension installed Passive (over days) Medium risk · extension-store review · privacy positioning
Recommended pattern Combine A + B in a single flow. OAuth login first. If the connected account is Business or Creator, pull recent media via API and build the style vector in 5 seconds. If the account is Personal (the majority), prompt the user to upload 5 photos — same 30-second onboarding. D is an add-on for the iOS / Android apps. C is excluded on TOS grounds.
Positioning vs Existing Categories
Catalog marketplaces (Farfetch, Wolf & Badger, Not On The High Street) sell inventory through search and browse. Affiliate networks (LTK, ShopMy) attach creator links to existing products. Creator-marketing platforms (Pearpop, TRIBE, Cohley) run campaigns for Fortune-500 brands at $50K+ budgets. None combine social discovery, visual AI and a reverse-marketplace structure inside one product, positioned for emerging affordable-luxury designers at $1K–$5K per ambassador deal.

Where the competitor map leaves a gap Every platform above runs the reverse-apply model for enterprise marketing budgets ($50K+ campaigns) or generalist DTC. None is built for emerging fashion designers selling drops at $1K–$5K of ambassador value per drop. None layers visual-AI style scoring on top of the brief-application flow. None ties the ambassador economics to a pre-order + co-creation marketplace. The reverse-apply mechanic is proven; the fashion-native, low-budget, AI-ranked, pre-order-integrated version is open territory.
