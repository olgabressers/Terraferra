Visual Discovery Engines
APIs TF can integrate to let consumers find products by image or aesthetic — pricing, integration cost, and honest assessment of what works at TF's stage.
Two different problems — make sure you're solving the right one
Problem A — "Show me similar products" on a product page. Consumer clicks a jacket, sees 6 similar jackets. This is behavioural recommendation (Algolia Recommend). It needs click and purchase history to train on — cold-start problem: zero data on day 1, gets better over time. Not true visual search.

Problem B — "Upload a photo, find matching products." Consumer uploads an Instagram screenshot, TF returns the closest items in its catalog. This is visual search (ViSenze, FashionCLIP). It does not need purchase history — it works from day 1 based on image similarity. This is what Maxim's comment is about: a new way to discover unusual items.

Algolia is cheap and easy but solves Problem A. ViSenze and FashionCLIP solve Problem B. They are not interchangeable.

02What a working integrated MVP actually requires
A working MVP means: consumer opens TF, uploads or takes a photo, sees matching products from TF's catalog. Not a widget on a product page — actual photo-to-catalog visual search.
Option A — FashionCLIP self-hosted (recommended if TF has engineering capacity)
  • 1
    Embed the TF product catalog. For each product image, run FashionCLIP to generate a 512-dimension vector. One-time batch job on a GPU instance. For 1,000–10,000 products: a few hours on AWS g4dn.xlarge spot instance. ~$1–5 one-time
  • 2
    Store vectors in Pinecone or Weaviate. Free tiers cover early-stage TF (Pinecone Starter: 2GB storage, 2M write units/mo; Weaviate: 100K objects). No cost until TF scales past these limits. $0/mo
  • 3
    Build a search API endpoint. Backend receives a consumer's uploaded image → runs FashionCLIP → queries vector DB → returns top-N product matches. Standard REST endpoint. 1–2 weeks engineering
  • 4
    Wire into the TF app UI. Add a camera/upload button. POST the image to the search API. Render the result grid. Works on both mobile app and web. 3–5 days frontend
Total cost: ~$5 one-time GPU + $0/mo ongoing at MVP scale. Time to working: 2–3 weeks with one backend/ML-capable engineer. Risk: Requires an engineer who can set up the ML pipeline — not a pure frontend task.
Option B — ViSenze 30-day trial (recommended if TF has no ML engineer)
  • 1
    Sign up for the ViSenze trial. No credit card. 30-day window, capped at 100K API requests and 50K products. Gives full production-grade visual search functionality during the trial. $0 during trial
  • 2
    Index TF's product catalog via ViSenze API. Upload product images and metadata. ViSenze handles embedding and indexing — TF engineering does not touch the ML layer. 2–3 days engineering
  • 3
    Integrate the visual search API into TF app. Camera/upload → POST to ViSenze API → render results. Same frontend work as Option A. 3–5 days frontend
  • 4
    Measure conversion during the 30-day window. If photo search converts, negotiate commercial pricing. If not, trial ends with no cost and data to show investors. But note: ViSenze is post-acquisition (Rezolve AI, Aug 2025) — confirm the trial is still active and what post-trial pricing looks like before starting.
Total cost: $0 during trial. Post-trial: custom quote only — no public pricing, could be expensive. Time to working: ~1 week. Risk: Unknown post-trial cost; vendor is mid-acquisition transition.
03Where Algolia fits — and what it does not do
Algolia Recommend is not a visual search engine. It belongs in TF's stack as a different layer.
Use case Algolia does this? Notes
"Show similar products" on product/drop page Yes "Looking Similar" feature. Starts working after TF accumulates click and purchase events. Weak on day 1 (cold start), stronger after weeks of data. Free up to 10K requests/mo.
"Upload a photo, find matching products" No Algolia does not accept image uploads as search queries. This requires ViSenze or FashionCLIP.
Text search ("flowy purple dress") Yes Core Algolia capability. Fast, well-documented.
Trending items / "What's popular now" Yes Trending Items model, included in Recommend tier.
Algolia's $0.60/1K recommend requests pricing is real — but at 10K free requests/month, a small TF MVP (few hundred DAU) will not hit the paid threshold for months. The free tier is genuinely usable. Add it to the product as a "you might also like" layer — it is not a substitute for photo-upload discovery.
